# AiBle BOX 신규 유틸리티 배포 프로세스 매뉴얼 (SOP)

> **전체 가이드:** [`AIBLE_BOX_유틸리티_배포_완전_가이드.md`](./AIBLE_BOX_유틸리티_배포_완전_가이드.md) (처음부터 끝까지, 환경 변수·코드·트러블슈팅 포함)

새 도구를 만들어 **aible-box.vercel.app/(슬러그)/** 하위 경로로 노출하기까지의 **표준 작업 순서**입니다.
복사-붙여넣기로 바로 적용할 수 있도록 코드 템플릿을 포함합니다.

> 실제 적용 예시: `admission-parser`(모집요강 학과 추출기) → `https://aible-box.vercel.app/admission/`

---

## 0. 아키텍처 한눈에 보기 (게이트웨이 패턴)

직원에게는 **aible-box 도메인 하나**만 알려줍니다. 각 도구는 **별도 Vercel 프로젝트**로 배포되지만,
aible-box의 `vercel.json` **rewrite**가 하위 경로 요청을 각 도구로 프록시합니다.

```
사용자 ──> https://aible-box.vercel.app/           (aible-box: 플랫폼 홈)
        ──> https://aible-box.vercel.app/chat/       (rewrite → seesign-chat.vercel.app)
        ──> https://aible-box.vercel.app/admission/  (rewrite → admission-parser.vercel.app)
```

| 프로젝트 | Vercel 배포 URL | 사용자에게 보이는 경로 | 슬러그 | 로컬 포트 |
|---|---|---|---|---|
| aible-box | aible-box.vercel.app | `/` | (게이트웨이) | 3000 |
| excel-ai-chat | seesign-chat.vercel.app | `/chat/` | `chat` | 3001 |
| admission-parser | admission-parser.vercel.app | `/admission/` | `admission` | 3002 |
| (다음 도구) | (도구).vercel.app | `/(슬러그)/` | (슬러그) | 3003 |

핵심 원리 3가지:
1. **각 도구는 `basePath`(예: `/admission`)로 빌드** → 모든 페이지/에셋/API가 그 경로 아래로 나옴.
2. **aible-box `vercel.json`이 그 경로를 도구 배포로 rewrite** → 같은 도메인처럼 보임.
3. **카드 링크는 절대 URL이 아니라 상대 경로**(`/admission/`)를 사용 → 인증 우회·CORS 방지.

## 1. 사전 결정 (새 도구마다 1회)

| 항목 | 정하는 법 | 예시 |
|---|---|---|
| 슬러그(slug) | 소문자+하이픈, 다른 도구와 중복 금지 | `admission` |
| 로컬 dev 포트 | 기존과 겹치지 않게 +1 | `3002` |
| Vercel 프로젝트명 | 보통 레포 이름 | `admission-parser` |

> 슬러그는 한 번 정하면 `basePath`, `vercel.json`, 카드 `href`, `middleware` matcher **네 곳에서 동일**해야 합니다.

## 2. 신규 도구 프로젝트 수정 (게이트웨이 대응 6단계)

`(슬러그)` 자리에 실제 슬러그를 넣으세요. 아래는 App Router + 루트 `app/` 기준 (src 미사용 시 `middleware.ts`는 프로젝트 루트).

### 2-1. `next.config.mjs` — basePath/assetPrefix

```js
/** @type {import("next").NextConfig} */
const basePath = process.env.NEXT_PUBLIC_BASE_PATH ?? "";

const nextConfig = {
  reactStrictMode: true,
  basePath: basePath || undefined,
  assetPrefix: basePath ? `${basePath}/` : undefined,
};

export default nextConfig;
```

### 2-2. `lib/base-path.ts` — 클라이언트 fetch 경로 헬퍼

```ts
export const basePath = (process.env.NEXT_PUBLIC_BASE_PATH ?? "").replace(/\/$/, "");

export function withBasePath(path: string): string {
  const normalized = path.startsWith("/") ? path : `/${path}`;
  return `${basePath}${normalized}`;
}
```

### 2-3. 모든 클라이언트 API 호출에 `withBasePath` 적용

```ts
import { withBasePath } from "@/lib/base-path";

// 변경 전: fetch("/api/parse/start", ...)
// 변경 후:
await fetch(withBasePath("/api/parse/start"), { method: "POST", body: form });
```
> `basePath`가 설정되면 API는 `/admission/api/...`로 서빙됩니다. 절대경로 `/api/...`로 호출하면 404가 납니다.

### 2-4. `middleware.ts` (프로젝트 루트) — 직접 접속 차단(선택)

```ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

function getAllowedHosts(): string[] {
  return (process.env.ALLOWED_HOSTS ?? "")
    .split(",").map((h) => h.trim().toLowerCase()).filter(Boolean);
}

export function middleware(request: NextRequest) {
  const allowed = getAllowedHosts();
  if (allowed.length === 0) return NextResponse.next(); // 미설정 시 통과
  const rawHost = request.headers.get("x-forwarded-host") ?? request.headers.get("host") ?? "";
  const host = rawHost.split(":")[0]?.toLowerCase() ?? "";
  if (!allowed.includes(host)) return new NextResponse("Forbidden", { status: 403 });
  return NextResponse.next();
}

export const config = {
  matcher: [
    "/(슬러그)",
    "/(슬러그)/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp)$).*)",
  ],
};
```

### 2-5. "← AiBle BOX" 복귀 버튼 — `Link`가 아닌 `<a>`

```tsx
const aibleBoxUrl = process.env.NEXT_PUBLIC_AIBLE_BOX_URL ?? "/";
// ...
<a href={aibleBoxUrl}>← AiBle BOX</a>
```
> Next `Link`를 쓰면 basePath 내부로만 이동합니다. 플랫폼 루트로 나가려면 일반 `<a>` + 절대 URL.

### 2-6. `package.json` dev 포트 + `.env.example`

```jsonc
// package.json
"dev": "next dev -p 3002",
"start": "next start -p 3002",
```

```bash
# .env.example
NEXT_PUBLIC_BASE_PATH=/admission
NEXT_PUBLIC_AIBLE_BOX_URL=http://localhost:3000   # 프로덕션: https://aible-box.vercel.app/
# ALLOWED_HOSTS=aible-box.vercel.app              # 커스텀 도메인 붙인 뒤 선택
# (도구 전용 API 키도 여기)
```

## 3. 신규 도구 Vercel 배포

1. GitHub에 푸시 → Vercel **New Project** → 레포 선택.
2. **Settings → Environment Variables**:
   | 변수 | 값 | 필수 |
   |---|---|---|
   | `NEXT_PUBLIC_BASE_PATH` | `/admission` | 필수 |
   | `NEXT_PUBLIC_AIBLE_BOX_URL` | `https://aible-box.vercel.app/` | 필수 |
   | (도구 API 키) | 예: `GEMINI_API_KEY`, `LLAMA_CLOUD_API_KEY` | 도구별 |
   | `ALLOWED_HOSTS` | `aible-box.vercel.app` | 나중에 |
3. Deploy 후 **단독 접속 테스트**: `https://(도구).vercel.app/admission` 가 정상 동작하는지 확인.
4. 배포된 실제 도메인을 메모(다음 단계 rewrite에 사용).

> API 키는 **각 도구 프로젝트에만** 넣습니다. aible-box에는 키를 넣지 않습니다.

## 4. aible-box(플랫폼) 수정 — 4곳

### 4-1. 카드 등록 — `src/data/utilities.ts`

```ts
{
  id: "admission-parser",
  name: "모집요강 학과 추출기",
  description: "대학 모집요강 PDF에서 전형별 학과리스트를 추출",
  features: ["...", "..."],
  href: process.env.NEXT_PUBLIC_ADMISSION_URL ?? "/admission/",  // 상대경로!
  tags: ["입시", "PDF"],
  category: "데이터 분석",
  developer: "심준혁",
},
```

### 4-2. 프로덕션 프록시 — `vercel.json` (rewrite 2줄)

```json
{ "source": "/admission",          "destination": "https://admission-parser.vercel.app/admission" },
{ "source": "/admission/:path*",   "destination": "https://admission-parser.vercel.app/admission/:path*" }
```
> `destination`은 **3단계에서 메모한 실제 도구 배포 도메인**으로 맞추세요.

### 4-3. 로컬 개발 rewrite — `next.config.ts`

```ts
const admissionOrigin = process.env.ADMISSION_DEV_URL ?? "http://localhost:3002";
// rewrites() 배열에 추가:
{ source: "/admission", destination: `${admissionOrigin}/admission` },
{ source: "/admission/:path*", destination: `${admissionOrigin}/admission/:path*` },
```

### 4-4. `.env.example` 문서화 + Vercel 환경변수

```bash
NEXT_PUBLIC_ADMISSION_URL=/admission/
```
| aible-box Vercel 변수 | 값 |
|---|---|
| `NEXT_PUBLIC_ADMISSION_URL` | `/admission/` |

## 5. aible-box 재배포 & 검증

1. aible-box를 Git push → Vercel **Redeploy** (rewrite는 재배포해야 적용됨).
2. 통합 접속 테스트:
   - `https://aible-box.vercel.app/` → 카드가 보이는가
   - 카드 클릭 → `https://aible-box.vercel.app/admission/` 진입하는가
   - 도구 내부 동작(업로드/API/다운로드) 정상인가
   - "← AiBle BOX" 클릭 시 플랫폼 홈으로 나가는가

## 7. 트러블슈팅

| 증상 | 원인 / 해결 |
|---|---|
| `/admission` 404 | 도구에 `NEXT_PUBLIC_BASE_PATH` 미설정 → 설정 후 Redeploy |
| CSS/JS 깨짐, 에셋 404 | basePath와 assetPrefix·rewrite·destination 슬러그 불일치 |
| API 호출 404 | 클라이언트 fetch에 `withBasePath` 누락 |
| 게이트웨이에서 rewrite 안 됨 | aible-box `vercel.json` 수정 후 **Redeploy** 안 함 |
| "← AiBle BOX"가 안쪽에서만 이동 | `Link` 대신 `<a href={NEXT_PUBLIC_AIBLE_BOX_URL}>` 사용 |
| 직접 도메인 접속을 막고 싶음 | 도구에 `ALLOWED_HOSTS=aible-box.vercel.app` 설정 |
